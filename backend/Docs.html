Secure Message API

OpenAPI spec: /openapi.json

Endpoints

  • GET /health
  • POST /api/register
  • POST /api/login
  • POST /api/logout
  • GET /api/me
  • GET /api/contacts
  • POST /api/contacts
  • DELETE /api/contacts/{alias}
  • GET /api/admin/users (admin only)
  • DELETE /api/admin/users/{username} (admin only)
  • GET /api/users/{username}/public-key
  • POST /api/messages
  • GET /api/messages (with, limit, order, before_id)

Protected endpoints require Authorization: Bearer <token>.

New registrations require a password with 8-128 characters, at least one number, and at least one special character.

The admin endpoints can list registered usernames and delete non-admin user accounts.

Message payloads are opaque encrypted blobs and may contain client-defined attachment envelopes for arbitrary files, including images.
